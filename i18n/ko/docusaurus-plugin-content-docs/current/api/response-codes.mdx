# 응답 코드 및 메시지

응답 코드는 6자리 길이입니다. 첫 번째 숫자는 요청이 실패했는지 여부를 나타냅니다.

* `0` - 요청이 오류없이 처리되었습니다.
* `1` - 요청이 실패했습니다.

다음 두 숫자는 오류 범주와 하위 범주를 나타내며 마지막 세 숫자는 이전 두 숫자가 나타내는 범주와 하위 범주에 대한 특정 오류 코드입니다.

아래의 예를 보십시오.

:::note
하십시오. 오류가 발생하면 반환된 HTTP 응답 코드는 요청이 실패하게 만든 오류(즉, `debug` 속성이 `false`로 설정된 경우 반환되는 유일한 오류)와 일치합니다.
:::

```
xxxxxx
││ └─ 특정 오류 코드, 3자리
│└─ 오류 카테고리 및 하위 카테고리, 2자리
│   ├─ 0 - 오류 아님
│   │  ├─ 00 - 특별한 메시지 없음
│   │  └─ 01 - 정보 메시지
│   ├─ 1 - 인증 오류
│   │  ├─ 11 - 인증 실패
│   │  └─ 19 - 기타 인증 관련 오류
│   ├─ 2 - 데이터 관련 오류
│   │  ├─ 21 - 예상된 데이터를 받지 못함
│   │  ├─ 22 - 받은 데이터의 형식이 잘못됨
│   │  ├─ 23 - 받은 데이터의 값이 유효하지 않음
│   │  ├─ 24 - 기존 데이터와 충돌
│   │  └─ 29 - 기타 데이터 관련 오류
│   ├─ 3 - 권한 관련 오류
│   │  ├─ 31 - 이 메서드를 허용하지 않음
│   │  ├─ 32 - 이 객체 또는 매개변수를 허용하지 않음
│   │  ├─ 33 - 전달된 값이 허용 범위를 벗어남
│   │  └─ 39 - 기타 권한 관련 오류
│   └─ 9 - 기타 오류
└─ 요청 실패 여부, 1자리
   ├─ 0 - 요청 성공
   └─ 1 - 요청 실패
```
