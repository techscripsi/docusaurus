# Response Codes and Messages


they might sometimes overlap with HTTP codes but it is consistent

```
x   xx   xxx
│   │     └─ specific error code
│   └─ error category and subcategory
│      ├─ 0 - no errors
│      │  ├─ 00 - no special messages
│      │  ├─ 01 - unused data accepted
│      │  └─ 09 - other messages
│      ├─ 1 - related to authentication
│      │  ├─ 11 - authentication failed
│      │  └─ 19 - other authentication related errors
│      ├─ 2 - data related
│      │  ├─ 21 - did not receive expected data
│      │  ├─ 22 - the received data has wrong format
│      │  ├─ 23 - the received data has invalid value
│      │  ├─ 24 - conflict with existing data
│      │  └─ 29 - other data related errors
│      ├─ 3 - permissions
│      │  ├─ 21 - not allowed to pass this method
│      │  ├─ 22 - not allowed to pass this object or parameter
│      │  ├─ 23 - the passed value is out of the allowed range
│      │  └─ 29 - other permission related errors
│      └─ 9 - other cases
└─ is the request successful
   ├─ 0 - success
   └─ 1 - failure
```

- 400 Bad Request
- 405 Method Not Allowed
- 409 Conflict
